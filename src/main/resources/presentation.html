Overview

프로젝트 개요 — 키오스크 중심

키오스크가 하는 일

  • 매장 방문 고객이 터치 / 브라우저로 직접 메뉴를 고르고, 장바구니·쿠폰·결제까지 한 흐름으로 처리합니다.
  • 회원 전화번호·포인트·쿠폰과 연동해 할인 반영과 적립 안내(스탬프 화면)를 제공합니다.
  • 주문이 DB에 쌓이면 관리자 화면에서 조리·픽업 상태를 관리할 수 있습니다.

브랜드 · 메인 화면 구성

  • 헤더에 카페 브랜딩(예: 버터벨 · BRUNCH & COFFEE)과 시계가 표시됩니다. (templates/kiosk/index.html + kiosk-header)
  • 주메뉴 탭: 「음식 메뉴」「음료 메뉴」 두 탭으로 나뉘며, 서버에서 카테고리 FOOD / DRINK 로 필터링합니다.
  • 메뉴 카드마다 가격·이미지·품절(available) 처리·담기 버튼이 있습니다.

대표 메뉴 라인업 (DB 샘플 기준)

브런치: 에그 베네딕트, 아보카도 토스트, 팬케이크, 샥슈카, 크로크 무슈, 어니언 수프 등

음료: 핫/아이스 아메리카노·라떼, 플랫 화이트, 카푸치노, 자몽 에이드, 딸기 스무디 등

관리자 서브시스템

  • 대시보드: 최근 주문·매출 요약
  • 메뉴 관리: 가격·설명·노출·이미지 경로
  • 주문 관리: 상태 변경·상세(줄 항목·결제)
  • 회원·쿠폰: 회원 조회 및 쿠폰 연계

Database

DB 테이블 (Supabase / PostgreSQL)

member

컬럼타입설명
idBIGSERIALPK
phoneVARCHAR(20)UNIQUE · NOT NULL
nameVARCHAR(100)이름
pointsINT포인트, 기본 0
created_atTIMESTAMP가입 시각

menu

컬럼타입설명
idBIGSERIALPK
nameVARCHAR(100)메뉴명 · NOT NULL
priceINT가격(원)
categoryVARCHAR(10)CHECK: FOOD / DRINK
image_urlVARCHAR(500)이미지 경로
descriptionVARCHAR(500)설명
availableBOOLEAN판매 여부
created_atTIMESTAMP

orders

컬럼타입설명
idBIGSERIALPK
member_idBIGINTFK → member
total_amountINT결제 총액
discount_amountINT할인액 기본 0
payment_methodVARCHAR(20)CARD 등
statusVARCHAR(20)RECEIVED / PREPARING / COMPLETED / CANCELLED
created_atTIMESTAMP

order_item

컬럼타입설명
idBIGSERIALPK
order_idBIGINTFK → orders · CASCADE
menu_idBIGINTFK → menu
quantityINT수량
unit_priceINT단가 스냅샷

주문 삭제 시 줄 항목도 함께 삭제되어 데이터 정합성을 유지합니다.

coupon

컬럼타입설명
idBIGSERIALPK
member_idBIGINTFK → member
codeVARCHAR(50)UNIQUE
discount_typeVARCHAR(10)FIXED / PERCENT
discount_valueINT정액(원) 또는 정률(%)
usedBOOLEAN사용 여부
expires_atTIMESTAMP만료
created_atTIMESTAMP

admin

컬럼타입설명
idBIGSERIALPK
usernameVARCHAR(50)UNIQUE
passwordVARCHAR(255)BCrypt 해시
created_atTIMESTAMP

ER 관점: 한 회원은 여러 주문·쿠폰을 가질 수 있고, 주문은 다수의 order_item으로 구성됩니다.

├── src/main/resources/
│   ├── application.yml                   ← DB URL, JPA, 로깅
│   ├── schema.sql                        ← Supabase DDL·샘플
│   ├── static/
│   │   ├── css/kiosk.css, admin.css
│   │   ├── js/kiosk.js, admin.js
│   │   └── images/ (menu, payment, logo…)
│   └── templates/
│       ├── fragments/layout.html
│       ├── kiosk/  (index, cart, payment, complete, stamp)
│       └── admin/   (login, dashboard, menus, orders, members)

관리자 — 사이트맵 · 업무 흐름

운영 흐름

로그인 성공 후 대시보드에서 당일 주문·상태를 파악하고, 메뉴 가격을 조정하거나 품절 처리합니다. 주문 화면에서 접수→조리→완료 등 상태를 바꾸고, JSON API로 상세(메뉴 줄·할인·결제수단)를 확인합니다. 회원 화면에서는 포인트·쿠폰 발급 이력을 함께 봅니다.

Git

Git 브랜치 전략 (트리)

버전은 2 ~ 4.1 구간으로 표기(세부 번호는 팀 내 스프린트·이터레이션용). 원격 origin에 동일 패턴이 있습니다.

main └── dev ◀── 기본 통합 브랜치 (origin/HEAD) ├── menu2 ~ menu4.1 ← 메뉴·키오스크 메인 ├── order2 ~ order4.1 ← 주문·결제·장바구니 ├── member2 ~ member4.1 ← 회원·쿠폰·스탬프 └── admin2 ~ admin4.1 ← Security·대시보드·통합·주문 관리 예: 현재 작업 admin4.1 에서 관리자·통합 이슈를 마무리한 뒤 dev 로 머지하는 식으로 운영.

정책 요약

main은 안정 릴리즈, 일상 통합은 dev. 기능 줄기는 도메인 이름 + 버전 레일(2·3·3.1·3.2·4·4.1)로 구분해 동시 작업 시 브랜치명만으로 역할을 알 수 있게 했습니다.
